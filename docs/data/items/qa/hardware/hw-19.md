# Q: 儲存空間顯示「系統資料 (System Data)」佔用了十幾甚至數十 GB，導致無法更新或變慢，怎麼辦？

# Answer

**這是 iPadOS 常見的快取堆積問題，通常源自於大量的 App 暫存檔、失敗的更新、**
**MDM 紀錄或一些未能正確刪除的檔案殘留。**

當「灰色」的系統資料佔用超過 15GB 甚至更多時，一般刪除 App 或照片已無濟於事，
必須透過「重置」來釋放空間。

**解決方案 (依序執行)**：

**1. 透過 MDM 遠端重置 (Wipe Device)**
*   若裝置已大量分發，管理員可透過 Jamf Pro 發送 **「清除裝置 (Wipe Device)」** 指令
    （強烈建議勾選 **Return to Service**），這是最乾淨的做法。

**2. 使用者自行操作 (若 MDM 未阻擋)**
*   若管理員在限制描述檔中**未勾選**「禁止清除所有內容和設定」，師生可自行操作。
*   **路徑**：前往 **「設定」>「一般」>「移轉或重置 iPad」>「清除所有內容和設定」**。
*   **注意**：此動作會將 iPad 恢復原廠設定，清除所有資料。

**實務建議：關鍵安全網**
無論是遠端清除還是手動清除，**務必確保該裝置已正確設定 ADE (自動裝置註冊) 與 PreStage Enrollment**。
*   **原因**：這樣裝置在重置後連上 Wi-Fi 時，才會自動跳出「遠端管理」畫面並重新接受 Jamf Pro 納管。
*   **風險**：若 ADE 設定有誤，學生手動清除後，該裝置可能會變成一台未受監管的「孤兒機」，導致管理中斷。
