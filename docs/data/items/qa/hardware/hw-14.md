## Q: 【網路隱私】為什麼在學校 Wi-Fi 的防火牆或 DHCP Server 上，抓不到 iPad 的真實 MAC 位址？

## Answer

**這是因為 iPadOS 預設開啟了「私密 Wi-Fi 位址 (Private Wi-Fi Address)」，**
**系統會針對不同的 SSID 生成虛擬的隨機 MAC 位址。**

這項功能旨在防止商業追蹤，但在校園網管中會造成災難。
若學校防火牆依賴「MAC Address Binding」來配發固定 IP 或控管上網權限，
隨機位址會導致 iPad 被視為陌生裝置而阻擋連線。

**解決方案 (MDM 統一派送)**：

請勿要求學生手動逐一關閉，應透過 Jamf Pro 統一處理：

1. **修改設定檔**：進入派送給學生的 **「Wi-Fi 設定描述檔」**。
2. **關鍵選項**：在 Wi-Fi 設定描述檔其 Payload 中，

 勾選 **「停用 MAC 位址隨機化 (Disable MAC Address Randomization)」**。

3. **效果**：當裝置連接該指定 SSID 時，將強制回報硬體真實的 MAC 位址，

 讓網管裝置能正確識別資產。
