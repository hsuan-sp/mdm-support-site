## Q: 收到 iOS 更新通知，我可以馬上更新嗎？還是要等學校通知？

## Answer

**建議等待學校通知後再更新。新版 iOS 剛推出時可能存在與教學 App 或 MDM 功能的相容性問題，**
**學校會在驗證後統一安排更新。**

Apple 每隔數週就會發布 iOS/iPadOS 更新，包含安全修補與新功能。
但對於教育環境而言，「最新」不一定等於「最適合」。

**為何不建議立即更新**：

| 潛在風險 | 說明 |
| :--- | :--- |
| **App 相容性問題** | 第三方教學 App 可能尚未針對新系統版本最佳化，導致閃退或功能異常。 |
| **MDM 功能異常** | 極少數情況下，新 iOS 版本與 Jamf Pro 的某些指令可能暫時不相容。 |
| **意外 Bug** | 新版本可能有 Apple 尚未發現的問題，數週後才會發布修補更新。 |
| **更新失敗風險** | 更新過程若中斷（如電量不足、網路斷線），可能導致裝置需要復原模式處理。 |

**學校的 N-1 原則**：

許多 IT 團隊遵循「N-1 原則」：讓生產環境（學生/教師裝置）保持在前一個穩定版本（N-1），
而非追求最新版本（N）。只有在：

* 新版本經過充分測試

* 關鍵 App 開發商確認相容

* 安全性更新為必要修補

時，才會推送更新。

**學校可能採取的更新策略**：

1. **延遲更新**：透過 MDM 設定「延遲軟體更新 X 天」，讓裝置不會立即收到更新通知。

1. **測試群組**：先對 IT 人員或自願者的裝置推送更新，驗證一週後再推送給全校。

1. **統一排程**：在非教學時段（如夜間或週末）透過 MDM 統一推送更新。

1. **封鎖特定版本**：若某版本有已知問題，可暫時封鎖。

**實務建議**：

* **不要輕易點選「立即安裝」**。若出現更新提示，選擇「稍後」或「今晚」

 （然後在設定中取消自動安裝）。

* **等待學校公告**。通常 IT 團隊會在驗證後發布「可以更新」的通知。

* **確保更新前電量充足（>50%）並連接穩定 Wi-Fi**。
