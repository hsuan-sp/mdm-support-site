# Q: 「引導使用模式 (Guided Access)」跟「單一 App 模式」有什麼不一樣？

# Answer

**兩者都能將 iPad 鎖定在單一 App 中，但控制權不同：**
**引導使用模式由使用者（老師）手動開關；單一 App 模式由 MDM 強制設定，重開機也無法解除。**

這兩種鎖定方式經常被混淆，但用途與彈性完全不同。

**功能比較**：

| 特性 | 引導使用模式 (Guided Access) | 單一 App 模式 (Single App Mode) |
| :--- | :--- | :--- |
| **設定者** | 使用者（老師、學生均可） | MDM 管理員 |
| **啟動方式** | 連按三次電源鍵/Home 鍵 | MDM 推送描述檔自動生效 |
| **解除方式** | 連按三次 + 輸入密碼 | 僅 MDM 移除描述檔 |
| **重開機後** | 解除（需重新啟動） | **仍被鎖定** |
| **適合情境** | 臨時性鎖定、單次考試 | 長期 Kiosk、嚴格考試環境 |
| **需 MDM** | ❌ 不需要 | ✅ 需要 |

---

**引導使用模式 (Guided Access)**

**啟用方式**：
1.  設定 > 輔助使用 > 引導使用模式 > 開啟。
2.  設定一組密碼（老師專用）。

**使用方式**：
1.  開啟目標 App。
2.  連按三次電源鍵（或 Home 鍵）。
3.  畫面會出現引導使用模式設定畫面。
4.  可進一步停用螢幕特定區域、硬體按鈕、觸控等。
5.  點選「開始」進入鎖定模式。
6.  學生無法退出該 App，除非連按三次並輸入正確密碼。

**優點**：靈活、即時、無需 IT 協助。
**缺點**：需逐台手動設定、學生若知道密碼可自行解除。

---

**單一 App 模式 (Single App Mode)**

**設定方式（Jamf Pro）**：
1.  建立設定描述檔，加入「單一 App 模式」Payload。
2.  輸入目標 App 的 **Bundle ID**（如：com.apple.Keynote）。
3.  設定 Scope 並推送。
4.  裝置收到描述檔後，會**立即**鎖定在該 App 中。

**解除方式**：
*   在 Jamf Pro 將該裝置從描述檔 Scope 中移除。
*   或刪除該描述檔。

**優點**：最強控制、無法繞過。
**缺點**：需管理員操作、不夠靈活。

---

**實務建議：使用情境**

| 情境 | 建議方式 |
| :--- | :--- |
| 課堂中的 10 分鐘小測驗 | 引導使用模式（老師手動啟動） |
| 正式期中/期末考試 | 單一 App 模式（由 MDM 統一設定） |
| 校慶活動展示機 (Kiosk) | 單一 App 模式 |
| 圖書館自助查詢機 | 單一 App 模式 |
| 臨時鎖定特定學生的 App | 引導使用模式 |

**Jamf Teacher 的課程鎖定**：

Jamf Teacher 提供介於兩者之間的方案：
*   老師可透過 Jamf Teacher App 建立「課程」，限制學生只能使用特定 App。
*   課程期間生效，結束後自動解除。
*   不需 MDM 管理員介入，老師可自行操作。
