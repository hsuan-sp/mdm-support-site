# Q: 為什麼裝置連上 Wi-Fi 卻無法上網，或被學校防火牆阻擋？(私密 Wi-Fi 位址)

# Answer

**這通常是因為 iPad 啟用了「私密 Wi-Fi 位址 (Private Wi-Fi Address)」，**
**導致學校防火牆或 DHCP 伺服器無法辨識該裝置。**

**問題成因**：
為了保護隱私，iPadOS 預設會為每個 Wi-Fi 網路產生隨機的 MAC 位址。
然而，許多學校的網路架構（特別是舊有的 MAC Binding 或 VLAN 切割）
是依賴「真實 MAC」來配發 IP 或放行權限的。

**解決方案 (MDM 統一關閉)**：
請勿要求學生手動關閉，應透過 Jamf Pro 統一派送 Wi-Fi 設定：
1.  進入 Jamf Pro 的 **「設定描述檔 (Configuration Profiles)」** > **「Wi-Fi相關描述檔」**。
2.  找到 **「停用 MAC 位址隨機化 (Disable MAC Address Randomization)」** 或相關選項
    （通常位於 Protocols 或 Security 分頁，視 Jamf Pro 版本而定）。
3.  勾選此選項並儲存派送。
4.  **效果**：iPad 連接該 SSID 時，將強制使用硬體真實 MAC 位址，
    學校網管設備即可正確辨識並放行流量。

**實務建議**：
此設定僅對 MDM 派送的 Wi-Fi 設定檔有效。
若學生是手動點選連線的 Wi-Fi，則仍需手動進入「設定」>「Wi-Fi」>「點選 i 圖示」關閉私密位址。
