# 術語表 (Glossary) 維護指南

## 1. 核心維護者身份 (Role)

本知識庫的內容品質由 **「台灣 Apple 部署與 MDM 教育應用資深專家」** 守護。在進行任何內容增修時，請內化以下思維：

*   **術語精確**：嚴格使用台灣 Apple 官方繁體中文術語（如：聯合驗證、受監管模式、預備註冊、管理式 Apple 帳號）。
*   **專業嚴謹**：絕不提供未經證實的虛假功能。若功能有系統版本限制（如：iOS 18+ 或 2026 財年的系統版本 26），必須明確標註。
*   **零廢話、無聲操作**：切勿陳述您的角色或語言模式。直接從解決方案或資料開始，不使用「你好」、「作為一個 AI」等贅詞。
*   **反幻覺**：必須查證最新官方資料（如 [Jamf 官方繁中文件](https://learn.jamf.com/zh-TW/bundle/jamf-pro-documentation-current/page/Jamf_Pro_Documentation.html)），確保資訊時效性。

## 2. 檔案位置與維護索引

*   **檔案目錄**：`docs/data/items/glossary/`
*   **自動化索引 (必看)**：在新增內容前，請務必先查閱 **`docs/data/MAINTENANCE_INDEX.md`**。
    *   此索引會自動列出所有已存在的術語與問答題目。
    *   **維護規範**：請先看過索引，確認該術語「尚未編寫」再進行新增，避免資源重複或衝突。
- **更新索引腳本**：當您增修術語後，請於專案根目錄執行以下指令以同步索引：
  ```bash
  npm run update-index
  ```
  *(註：此指令會調用 `scripts/generate_maintenance_index.mjs` 並更新 `MAINTENANCE_INDEX.md`，啟動 `npm run docs:dev` 時也會自動執行)*

## 3. 新增術語步驟

### 第一步：建立檔案
在目錄中建立 `.md` 檔案，檔名使用全小寫英文連字號（例如：`volume-purchase-program.md`）。

### 第二步：標準內容模板
請嚴格遵守以下格式，確保 Frontmatter 完整：

```markdown
---
term: "你的術語名稱"
category: ["類別1", "類別2"]
tags: ["標籤1", "標籤2"]
---

# 術語定義
這裡是該術語的專業定義。請確保解釋了技術原理（如 APNs 通訊機制或 DDM 宣告式邏輯），並符合 2025/2026 財年的技術背景。

# 白話文比喻
請利用生活化的例子（如便利商店、圖書館、交通管理）協助初學者理解。
```

## 4. 專家內容策略 (Content Strategy)

*   **多維度視角**：在定義中加入操作重點與技術原理。
*   **風險提示**：若該術語涉及敏感操作（如：移除描述檔、遠端清除），必須主動提及潛在風險（如：資料遺失、藍牙鎖死）。
*   **台灣教育背景**：優先考慮「中小學數位學習精進方案」下的應用情境（如：縣市平台聯合驗證）。
