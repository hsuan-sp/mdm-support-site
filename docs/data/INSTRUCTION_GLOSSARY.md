# 術語表 (Glossary) 維護指南

本文件說明如何新增、修改或刪除術語表項目。本系統採用「原子化」結構，每個術語為獨立的 Markdown 檔案。

## 1. 檔案位置
所有術語檔案位於：
`docs/data/items/glossary/`

## 2. 新增術語步驟

### 第一步：建立檔案
在上述目錄中建立一個新的 `.md` 檔案。
**檔名規則**：請使用全小寫英文，單字間以連字號 `-` 分隔。
- 例如：`volume-purchase-program.md` 或 `vpp.md`
- 若包含特殊符號請省略。

### 第二步：填寫內容
請複製以下模板並填入內容：

```markdown
---
term: 你的術語名稱 (例如：Volume Purchase Program)
category: 類別 (例如：Apple, Jamf, Network, Security)
tags: [相關標籤1, 相關標籤2]
---
# 術語定義
這裡是該術語的專業定義，請使用專業、準確的語言描述。

# 白話文比喻
這裡是該術語的白話文比喻。
請想像你正在跟一個完全不懂 IT 的人解釋，使用生活化的例子（例如：便利商店、圖書館、交通工具）。
```

## 3. 索引與同步
本網站設有 **自動索引機制** (`docs/data/all-data.data.ts`)。
- **您不需要手動更新任何 JSON 索引檔。**
- 只要新增或修改 Markdown 檔案，系統在下一次建置 (Build) 或開發伺服器重啟時，就會自動抓取最新資料。
- 排序邏輯：系統會自動根據 `term` 欄位進行英文字母排序。

## 4. 注意事項
- **Frontmatter (--- 包夾的區域)** 是必須的，請勿刪除。
- `term` 欄位決定了顯示在網站上的標題。
- 內容請保持「定義」與「比喻」雙段式結構，這對於一般使用者理解非常有幫助。
