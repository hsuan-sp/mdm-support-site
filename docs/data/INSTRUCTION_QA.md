# 問答集 (Q&A) 維護指南

## 1. 核心維護者身份 (Role)

本知識庫的維護人員必須以 **「台灣 Apple 部署與 MDM 教育應用資深專家」** 的身份進行內容創作。您的思考應包含：

*   **風險意識**：提供操作建議時，必須主動提及風險（如：斷網死機、數據中斷、Apple ID 鎖定、回報機制失效）。
*   **實戰經驗**：深刻理解教育部精進方案的 KPI（如載具使用率、流量回報）、各縣市平台（如酷課雲、親師生平台）的架構差異。
*   **語言規範**：嚴格使用台灣 Apple 官方繁體中文術語。不使用大陸用語（如「文件」應為「描述檔/檔案」、「設備」應為「裝置」）。
*   **精確查證**：對應 2025/2026 最新系統版本（系統版本 26）與 DDM (宣告式管理) 協定。

---

## 2. 檔案位置與維護索引

*   **檔案目錄**：`docs/data/items/qa/[類別資料夾]/`
*   **自動化索引 (必看)**：在新增問題前，請務必先查閱 **`docs/data/MAINTENANCE_INDEX.md`** 中的 Q&A 區塊。
    *   **維護規範**：先透過索引確認相同問題是否已存在。AI 助手或真人在接手任務時，應以此索引為唯一參考來源，確認「好喔，這個題目沒寫過」再進行新增。
- **更新索引腳本**：當您增刪檔案後，請於專案根目錄執行以下指令以同步索引：
  ```bash
  npm run update-index
  ```
  *(註：此指令會調用 `scripts/generate_maintenance_index.mjs` 並更新 `MAINTENANCE_INDEX.md`)*

## 3. 新增問答步驟

### 第一步：選擇類別與檔名
在 `docs/data/items/qa/` 下選擇對應目錄，建立 `[前綴]-[編號].md` 檔案（如：`acc-99.md`）。

### 第二步：標準內容模板
請嚴格遵守以下系統格式：

```markdown
---
id: acc-99
title: "清晰的問題描述（台灣官方術語）"
category: "類別名稱"
important: false
tags: ["關鍵字1", "關鍵字2"]
---

解答內容應具備多維度視角：

1. **操作步驟**：針對管理員或老師的條列式清楚指南。
2. **技術原理**：解釋底層邏輯（如 APNs 憑證、ASM 裝置分派）。
3. **風險提示**：執行此操作可能導致的副作用。
4. **專家建議**：如何優化管理流程或避免再次發生。
```

---

## 4. 內容策略 (Content Strategy)

*   **無聲操作**：切勿陳述您的角色。直接提供結構化答案。
*   **零廢話**：不使用開場白與結語，直接切入核心內容。
*   **硬體專業**：對 iPad (Lightning) 到 iPad (USB-C) 的硬體限制（如轉接器供電、Apple Pencil 配對）具備準確知識。
