# 問答集 (Q&A) 維護指南

本文件說明如何新增、修改或刪除 Q&A 項目。每個問題都是一個獨立的 Markdown 檔案。

## 1. 檔案位置
所有問答檔案依類別存放於：
`docs/data/items/qa/[類別資料夾]/`

目前的類別資料夾包括：
- `account` (帳號與伺服器)
- `enrollment` (裝置註冊)
- `apps` (App 管理)
- `classroom` (課堂管理)
- `digital-learning` (數位精進)
- `hardware` (硬體排除)
- `mac` (Mac 管理)
- `qa-education` (教育實戰)

## 2. 新增問答步驟

### 第一步：選擇類別與檔名
進入對應的類別資料夾，建立新的 `.md` 檔案。
**檔名規則**：`[類別前綴]-[編號].md`
- 例如：`acc-99.md`, `enr-50.md`
- 建議編號接續該類別目前最大的編號，以免重複。

### 第二步：填寫內容
請複製以下模板並填入內容：

```markdown
---
id: acc-99  <-- 請確保此 ID 與檔名對應且在該類別中唯一
title: 這裡是問題主旨 (例如：如何重置密碼？)
category: 帳號與伺服器 <-- 顯示在卡片上的分類名稱
important: false <-- 是否為重點問題 (true/false)
tags: [密碼, 重置, 帳號]
---
這裡是問題的詳細解答。
可以使用 Markdown 語法，例如：
1. 第一步
2. 第二步
   - 子步驟
   
> 補充說明或注意事項
```

## 3. 索引與同步
本網站設有 **自動索引機制** (`docs/data/all-data.data.ts`)。
- **您不需要手動更新任何 JSON 索引檔。**
- 只要檔案位於正確的資料夾內，系統在 Build 時會自動掃描並加入搜尋索引。
- 排序邏輯：系統會自動根據 `id` 欄位進行排序 (例如 `acc-01` 會排在 `acc-02` 前面)。

## 4. 注意事項
- **ID 必須唯一**：每個問題的 `id` 欄位是用來排序和識別的關鍵，請勿重複。
- **Tags**：有助於使用者透過關鍵字搜尋找到此問題，請盡量提供相關的關鍵字。
