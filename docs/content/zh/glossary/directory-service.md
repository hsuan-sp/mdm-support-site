## 術語定義

**Directory Service（目錄服務）** 是一種集中式系統，用來儲存、組織與管理組織內的身分資訊，例如使用者、裝置、群組以及存取權限。

它是組織中判定以下資訊的**權威來源**：

- 使用者是誰
- 隸屬於哪個單位或部門
- 可以存取哪些系統與資源

### 常見實作

- **Microsoft Active Directory（AD）**
- **LDAP（Lightweight Directory Access Protocol）**
- **雲端身分服務**
  - Azure AD（Entra ID）
  - Google Workspace
  - Okta

### 與 MDM 的整合

當 **行動裝置管理（MDM）** 與目錄服務整合後，使用者可以透過既有的組織帳號註冊裝置並存取內部資源。

這樣的整合可實現：

- 集中式身分驗證
- 依據角色或群組自動派發 App 與設定
- 使用者離職或離校時，即時回收所有受管裝置的存取權限

## 白話文比喻

目錄服務就像組織的 **「中央名冊」**。

它保存了每一位成員的正式紀錄：是誰、屬於哪個部門、擁有哪些權限。

當使用者嘗試登入 Wi-Fi、Email，或進行 MDM 裝置註冊時，  
這些系統都會查詢這個中央名冊來確認身分與權限，確保存取控管一致，並防止未授權的人員進入內部系統。
