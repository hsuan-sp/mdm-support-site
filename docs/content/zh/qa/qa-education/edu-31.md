**兩者在功能權限、更新頻率與「全域功能開關控制權」上存在差異，但資產序號可以在不同伺服器間自由轉移。**

## 帳號權限三層級

在「教育部版 Jamf Pro」中，權限依行政與技術安全需求分為三種角色：

1.  **教育部管理人員 (Full Admin)**：擁有最高管理權限。負責全台統一的安全政策制定、全域功能開關（如跨網段 DDM 選項）。
1.  **經銷商/廠商 (MSP)**：具備多校大規模部署權限，輔助學校進行初期的 ASM 設定與註冊。
1.  **校方管理帳號 (Site Admin)**：擁有日常維運權限（如 App 指派、智慧群組、裝置重新命名）。但在政策安全性上有保護限制，例如：**無法刪除核心描述檔**，以防止重要資安設定被誤移除。

## 核心差異比較

| 項目               | **教育部版 (校方帳號)**           | **校方自購版 (Full Admin)**         |
| :----------------- | :-------------------------------- | :---------------------------------- |
| **最高管理權限**   | **教育部核心團隊**                | **校方 100% 自主掌控**              |
| **描述檔管理**     | **受保護權限** (避免資安政策斷層) | **完全權限** (可自由增刪測試)       |
| **軟體更新 (DDM)** | 依中央統一排程與開放程度為準      | 可自行設定強制更新死線 (Deadline)   |
| **自動化與 API**   | 僅開放維運 API                    | 完全開放 (支援自訂 Python/Zsh 腳本) |
| **功能更新速度**   | 需經穩定度評測後由中央統籌開啟    | 官方釋出後隨時可自行啟用新功能      |

## 為什麼功能看得到卻不能點？

資訊組長在後台若發現某些新功能（如 iOS 26 的 PSSO 負載或特定 AI 控管按鈕）無法操作，主因如下：

- **權限層級限制**：部分功能需要全域 (Global) 層級的開關，這僅限教育部 Full Admin 權限者可見並勾選。
- **行政維穩優先**：為了確保全台海量設備的基礎教學環境不被非預期的新功能設定干擾，中央通常會延後開啟實驗性功能。
