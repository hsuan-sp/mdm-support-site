## 1. Frontmatter 規範

### QA 檔案

```yaml
---
category: 註冊與自動派送 (Enrollment)  # 必須，對應分類索引
id: enr-25                               # 必須，唯一識別碼
important: false                         # 必須，是否為重要題目
tags:                                    # 必須，至少一個標籤
  - ADE
  - 設定助理
  - 部署策略
title: ADE 自動註冊時，IT 應該「跳過」哪些設定助理步驟？  # 必須
---
```

### 術語檔案

```yaml
---
category:                                # 必須，可為陣列
  - Enrollment
term: ADE (Automated Device Enrollment)  # 必須，術語名稱（含英文縮寫）
---
```

**規則**：
- Frontmatter 中的鍵必須依「字母順序」排列
- `category`, `id`, `title`/`term` 為必填欄位
- 所有 frontmatter 值結尾**不可有多餘空格**

## 2. 標題結構

### QA 檔案標題層級

```markdown
正文內容...

### 子標題（如有需要）

詳細說明...
```

**規則**：
- 問題必須用 `## Q: ` 開頭
- 答案必須用 `## Answer` 標示
- 子標題使用 `###`，最多到 `####`

### 術語檔案標題層級

```markdown
## 術語定義

術語的完整定義...

## 白話文比喻

簡單易懂的日常類比...

## MDM 管理情境 (MDM Context)

（選用）管理層面的補充說明...
```

**規則**：
- 中文檔案使用「術語定義」與「白話文比喻」
- 英文檔案使用「Term Definition」與「Analogy」
- 標題文字固定，不可更改

## 3. 排版規則

### 空行規範

```markdown
## 標題

段落一的內容。

段落二的內容。
```

**規則**：
- 標題後必須有**一個空行**
- 段落之間必須有**一個空行**
- 清單前後必須有**一個空行**
- **禁止連續三個以上空行**

### 程式碼區塊

```markdown
段落內容。

` ``javascript
const example = 'code';
` ``

後續段落。
```

**規則**：
- 程式碼區塊前後必須有**一個空行**
- 程式碼區塊必須標註語言名稱（如 `javascript`, `bash`）
- 行內代碼使用單個反引號：`` `code` ``

## 4. 清單格式

### 無序清單

```markdown
- **主要項目**：說明文字。
- **次要項目**：說明文字。
  - 縮排子項目
  - 另一個子項目
```

**規則**：
- 使用 `*` 作為清單標記（不使用 `-`）
- 清單符號後必須有**一個空格**
- 子清單縮排**兩個空格**

### 有序清單

```markdown
1. **第一步**：詳細說明。
2. **第二步**：詳細說明。
3. **第三步**：詳細說明。
```

**規則**：
- 數字序號使用 `1.`, `2.`, `3.` 格式
- 序號後必須有**一個空格**

## 5. 粗體與強調

### 粗體語法

```markdown
這是 **粗體文字** 的範例。
```

**規則**：
- 使用 `**text**` 標記粗體（不使用 `__text__`）
- 粗體標記前後必須有**一個空格**（除非緊鄰標點符號）
- **禁止** `**text**` 這種內部有空格的寫法
- **禁止** `**text**word` 這種粘連寫法

### Emoji 使用

```markdown
- **✅ 建議做法**：說明。
- **❌ 禁止做法**：說明。
- **⚠️ 注意事項**：說明。
```

**規則**：
- Emoji 後必須有**一個空格**
- 常用 Emoji：✅（正確）、❌（錯誤）、⚠️（警告）、💡（提示）

## 6. 程式碼與專有名詞

### 行內代碼

```markdown
使用 `MDM` 指令來管理裝置。
```

**規則**：
- 技術指令、函數名、變數名使用反引號包裹
- 中文與反引號之間必須有**一個空格**

### 專有名詞

```markdown
**Apple School Manager (ASM)** 是學校專用的管理平台。
```

**規則**：
- 首次出現的專有名詞使用粗體，並附上英文全稱
- 縮寫與中文名稱保持一致：`ASM (Apple School Manager)` 或 `Apple School Manager (ASM)`

## 7. 中英文混排

### 盤古之白

**繁體中文檔案適用**：中文與英文/數字之間必須有一個空格。

```markdown
✅ 正確：這台 iPad 支援 iOS 26 系統。
❌ 錯誤：這台iPad支援iOS 26系統。
```

**規則**：
- 中文字元與英文字母之間：**加空格**
- 中文字元與阿拉伯數字之間：**加空格**
- 英文字母與中文標點符號之間：**不加空格**

示例：
```markdown
✅ Apple 於 2024 年推出了 iOS 18。
❌ Apple於2024年推出了iOS 18。
```

### 英文檔案規範

**英文檔案**：遵循標準英文排版規則。

```markdown
✅ Correct: The iPad supports iOS 26.
❌ Wrong: TheiPad supportsiOS26.
```

**規則**：
- 英文單詞之間必須有空格
- 專有名詞首字母大寫
- 句子結尾使用英文標點符號

## 8. 自動化工具

### 自動格式化腳本

專案提供統一的格式化工具：

```bash
npm run fix-markdown
```

**功能**：
- 自動修正粗體、Emoji、盤古之白等排版問題
- 標準化 frontmatter 順序
- 清除多餘空行
- **羃等性保證**：重複執行不會破壞已正確的格式

### 檢查腳本

```bash
npm run update-index
```

**功能**：
- 檢查所有 Markdown 檔案的 frontmatter 完整性
- 更新維護索引檔案

## 📝 完整範例

### QA 檔案範例

```markdown
---
category: 註冊與自動派送 (Enrollment)
id: enr-25
important: false
tags:
  - ADE
  - 設定助理
title: ADE 自動註冊時，IT 應該「跳過」哪些設定助理步驟？
---

自動裝置註冊 (ADE) 允許 IT 預先設定並跳過 iPad 或 Mac 初始設定時的多個冗長畫面。

### 建議「跳過」的步驟

- **✅ App 分析**：由學校代表組織決定即可。
- **✅ 螢幕使用時間**：學校應使用 MDM 限制設定檔來管理。
  - 避免學生自行設定
  - 統一由 IT 部門管控

### 必須「顯示」的步驟

- **❌ 遠端管理**：無法跳過，這是 ADE 核心步驟。
```

### 術語檔案範例

```markdown
---
category:
  - Enrollment
term: ADE (Automated Device Enrollment)
---

## 術語定義

- **Automated Device Enrollment (ADE)** 是大規模部署 Apple 裝置的黃金標準（前身為 **DEP**）。

運作流程包括：

1. **採購**：透過授權經銷商或 Apple 直接購買裝置。
2. **指派**：序號會自動加入組織的 **ASM (Apple School Manager)** 帳戶。
3. **啟用**：裝置開機並連上 Wi-Fi 時，自動下載 MDM 描述檔。

## 白話文比喻

這是裝置的 **「出廠身分綁定」**。

就像每一台學校的 iPad 在出生時，數位額頭上就被蓋了學校的鋼印。
```
