**iOS 26、iPadOS 26 與 macOS 26 引入的「Network Extension URL Filtering API」是內容過濾技術的技術升級選項。建議學校了解這項發展，但需注意精進方案的載具仍優先遵循教育部規範的「內容過濾 (Content Filter)」與「DNS Proxy」架構運作。**

## 內容過濾技術的演進 ：

| 時期            | 技術方式          | 優點                     | 可能的限制                             |
| :-------------- | :---------------- | :----------------------- | :------------------------------------- |
| **iOS 15 以前** | Per-App VPN       | 技術成熟                 | 狀態列顯示 VPN 圖示；可能較耗電        |
| **iOS 16-25**   | DNS Proxy         | 無 VPN 圖示；輕量化      | 僅能過濾網域名稱，無法精確過濾完整 URL |
| **iOS 26+**     | URL Filtering API | 效能優化；省電；精確過濾 | 需要 iOS 26 或更新版本                 |

## iOS 26 URL Filtering API 的技術特性 ：

### 1. 更精確的過濾能力

- **過去限制** ：只能辨識網域（如 `example.com`），同網域下所有內容都會被同樣處理。
- **新技術優勢** ：可識別完整 URL 路徑（如 `example.com/inappropriate-content`）。
- **實際應用** ：可封鎖特定頁面，而保留同網域的其他正常內容。

### 2. 隱私友善的設計

- **裝置端判斷** ：大部分過濾判斷在裝置上透過 Bloom Filter 完成，不需將完整瀏覽記錄傳送至雲端。
- **匿名查詢機制 (PIR)** ：當需要雲端資料庫核對時，採用「私密資訊檢索 (Private Information Retrieval, PIR)」技術，伺服器無法得知裝置具體查詢了哪個網址。
- **雙重隱秘傳輸** ：透過 Oblivious HTTP (OHTTP) 中繼轉發，確保 Apple 或服務提供商皆無法將查詢內容與特定使用者身分關聯。
- **不解密內容** ：僅檢查網址資訊，不會且無法解密 HTTPS 連線內容。

### 3. 效能與使用體驗優化

- **系統層級整合** ：由作業系統底層直接處理，效率更高。
- **背景運作** ：不會在畫面上顯示額外圖示或提示。
- **電池續航** ：建議相較舊架構更省電。

## 對學校的可能影響 ：

### 1. 教育平台相容性

- **可能的改善** ：
  - 新技術建議能更準確識別教育部與縣市教網中心的教學平台。
  - 假設能減少數位教科書或教學影片被誤判為需要過濾的內容。
  - 建議內容過濾廠商可建立更精確的教育平台允許名單。

- **學校可以做的** ：
  - 觀察更新後教學平台是否運作更順暢。
  - 若仍有誤擋情況，建議透過服務廠商回報。

### 2. 管理設定調整

- **好消息** ：
  - 假設使用 Jamf Trust 或類似服務，建議會自動切換至新技術。
  - 學校 IT 管理者通常無需手動修改設定。
  - 舊版 iOS 裝置建議仍可使用原有技術。

- **建議注意** ：
  - 裝置更新至 iOS 26 後，觀察過濾效果是否正常。
  - 假設發現異常，可先檢查 MDM 設定是否正確。
  - 必要時可聯繫內容過濾服務提供商確認。

### 3. 過濾效果評估

- **可能的優勢** ：
  - 建議誤擋率可能降低（正常教學內容較不易被錯誤封鎖）。
  - 建議覆蓋率可能提升（能處理更複雜的網址結構）。
  - 假設對加密網站的過濾效果可能更好。

- **評估方式** ：
  - 詢問教師在使用教學網站時是否遇到阻擋。
  - 觀察學生使用 iPad 時是否能正常存取教學資源。
  - 假設有內容過濾報表，可比較更新前後的誤擋情況。

## 給學校的建議 ：

### 系統更新策略

- **建議考量因素** ：
  - iOS 26 提供更好的內容過濾技術。
  - 假設希望獲得最佳過濾效果，建議逐步更新裝置。
  - 假設擔心相容性，建議可先在少數測試機上驗證。

- **更新時機建議** ：
  - 假設決定更新，建議選擇學期初或寒暑假進行。
  - 建議預留時間觀察更新後的運作狀況。
  - 假設發現問題，保留部分未更新裝置作為備用。

### 內容過濾服務評估

- **假設學校正在使用內容過濾服務** ：
  - 建議了解服務提供商是否已支援新 API。
  - 假設服務已停止或到期，評估是否需要繼續訂閱。
  - 建議基本的不當內容過濾功能通常仍建議保留。

- **假設學校未使用內容過濾服務** ：
  - 評估是否有保護學生網路安全的必要性。
  - iOS 26 的新技術讓內容過濾更有效率。
  - 可諮詢其他學校的使用經驗作為參考。

### 技術支援準備

- **教師說明** ：向教師說明這是技術升級，可能改善教學平台使用體驗。
- **問題回報** ：建立管道讓教師回報網站無法存取的問題。
- **測試流程** ：建議建立常用教學網站清單，更新後進行測試。
- **備案準備** ：假設遇到嚴重問題，準備暫時調整過濾設定的方案。
